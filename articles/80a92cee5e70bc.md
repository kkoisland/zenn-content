react-confetti は、紙吹雪アニメーションを React で簡単に実装できるライブラリです。gravity（重力）や wind（風）など複数のパラメーターがあり、数値を変えるだけでアニメーションの雰囲気がかなり変わります。

「動かしながら試せるツールがあればいいな」と思って Playground を作りました。

## 作ったもの

デモ: https://www.kkoisland.com/react-confetti-playground/
GitHub: https://github.com/kkoisland/react-confetti-playground

![React Confetti Playground](https://www.kkoisland.com/react-confetti-playground/ogp.png)

スライダーを操作するとリアルタイムにアニメーションが変わります。設定したパラメーターは `<Confetti />` のコードスニペットとしてそのままコピーできます。

## 機能

- 7つのパラメーターをスライダーで調整（numberOfPieces, gravity, wind, friction, opacity, initialVelocityX, initialVelocityY）
- カスタムカラー設定（最大5色）
- 各パラメーターの個別リセット・カラーリセット・全体リセット
- コードスニペット自動生成 & ワンクリックコピー
- 5つの季節プリセット（Sakura 🌸、Snow ❄️、Autumn 🍁、Star ✨、Christmas 🎄）
- 10言語対応（EN, JA, ZH, KO, ES, FR, DE, IT, NL, SV）
- ダークモード対応・レスポンシブ対応

## 技術スタック

React 19 + TypeScript + Vite + Tailwind CSS 4 + react-confetti 6.4.0 + react-intl（i18n）

## プリセット紹介

5つのプリセットは私が作りました。それぞれ季節感が出るようにパラメーターを調整しています。

- **Sakura 🌸** — `wind: 0.01` の微風で花びらがゆれながら、`gravity: 0.02` でゆっくり落ちます
- **Snow ❄️** — 風なしで静かに降ります。`gravity: 0.01` のかなりゆっくりな落下で、`initialVelocityY` でウェーブを防いでいます（ダークモード推奨）
- **Autumn 🍁** — `wind: 0.02` と `gravity: 0.08` で、落ち葉が風に舞いながら速めに落ちます
- **Star ✨** — `gravity: 0.0005` の超低重力。降ってきた粒子がほぼ止まって漂います。金・銀・薄めのゴールドの3色（ダークモード推奨）
- **Christmas 🎄** — `initialVelocityX: 5` で横に広がりながら落ち、華やかさを演出します

## こんなふうにもできる

- **Snow や Sakura に wind を足す** — 吹雪や桜吹雪のようになります
- **色を変えると雰囲気が変わる** — Autumn に黄色や黄緑を足すと紅葉らしい色合いに、Christmas を青と白だけにすると冬らしい印象に変わります（最大5色まで設定可能）
- **opacity を下げる** — どのプリセットでも透け感が出て、幻想的な雰囲気になります

プリセットをベースにしなくても、もちろん全パラメーターをゼロから組み合わせてオリジナルの演出を作ることもできます。

## 作っていて気づいたこと

### canvas は position: absolute（デフォルト）

react-confetti v6.4.0 のデフォルトは `position: absolute` です。ページをスクロールすると canvas ごと上に流れてしまい、紙吹雪が見えなくなります。

```tsx
// style を明示的に指定するだけで解決
<Confetti
  style={{ position: "fixed" }}
  // ...
/>
```

ドキュメントには書かれていないので、スクロール付きのレイアウトに組み込む場合は注意が必要です。

### 低重力で紙吹雪がウェーブ状に出る現象

`gravity` を低く設定すると（0.02 以下くらい）、紙吹雪が「大量 → 途切れ → 大量」というサイクルを繰り返します。

react-confetti は起動時に `numberOfPieces` 個の粒子を一斉に生成します。gravity が高いと粒子が速く落ちてバラバラのタイミングで画面外に出るため、補充もばらけて均一な密度を保てます。しかし gravity が低いと粒子がゆっくり落ちるため、ほぼ同時に生成された粒子がほぼ同時に画面外に出て、一斉にリサイクルされます。この「同期したリサイクル」がウェーブを生んでいるようです。

### initialVelocityY で緩和できる

`initialVelocityY` に負の値を設定すると、粒子はビューポートの上端よりさらに上の見えない領域に向けて打ち出されます。この「見えない上昇フェーズ」の間、粒子はまだ画面に登場していません。重力に引き戻されてビューポートに入ってきたとき、初めて紙吹雪として見えるようです。

値の大きさは粒子ごとにランダムに変化するため、それぞれが見えない領域に滞在する時間がバラバラになり、これが「画面への登場タイミングのばらつき」を生み、一斉リサイクルの同期を崩しているようです。見た目の動きはつねに下向きで、上向きに飛ぶ様子は一切見えません。

Playground では Snow に `-7`、Star に `-9` を設定しています。
