**stable-context-map** は、検索しても地図のビューポートが動かない地図アプリです。作ってから毎日のように使っています。

通常の地図アプリでは、地名を検索するたびに地図が移動します。京都エリアを調べようと縮尺を調整しても、「清水寺」を検索すれば清水寺にズームインし、「金閣寺」を検索すれば金閣寺に移動します。こういったビューポートのリセットがない地図アプリを作りたいと考え、完成しました。

- ライブデモ: https://www.kkoisland.com/stable-context-map/
- GitHub: https://github.com/kkoisland/stable-context-map

## アーキテクチャについて

このアプリはAPI Keyなしで動きます。OpenStreetMap / react-leaflet / Nominatim の構成については、別の記事で整理しています。

https://zenn.dev/kkoisland/articles/043c648822dad3

## 設計の核心：center を検索で更新しない

通常の地図アプリでは、検索後に `map.setView()` や `map.flyTo()` で地図を移動させます。stable-context-mapではそれをしません。検索結果はマーカーとして追加するだけで、地図の位置は変えません。`center` は React の state で持っていますが、検索のたびに更新しません。

```ts
const handleSearch = async (query: string) => {
  const result = await searchLocation(query);
  const newMarker = createMarker(Number(result.lat), Number(result.lon), query);

  // 地図は動かさず、markerを追加するだけ
  setMarkers((prev) => [...prev, newMarker]);
};
```

ただし、最初のマーカー追加時だけは例外です。「どのエリアを調べるか」というエリア設定として機能するからです。2件目以降は地図を動かしません。

```ts
if (markers.length === 0) {
  setCenter([newMarker.lat, newMarker.lng]);
}
setMarkers((prev) => [...prev, newMarker]);
```

## できること

- 🔍 検索しても地図が動かない（検索結果はマーカーとして追加される）
- 📍 地図上のクリックでもマーカーを追加できる
- マーカーにラベルとメモを編集できる
- マーカー一覧を右上に表示。選択・削除・地図上の場所へのジャンプができる
- 🌍 全マーカーが収まる範囲に自動調整（Fit Bounds）
- 🔒/🔓 ズームロックで誤操作を防止
- 📏 ズームセレクターで16段階のプリセットから選択
- 💾 localStorageに自動保存
- 📥 📤 JSONでインポート／エクスポート
- 🖨️ PDFで書き出し

以下で各機能を説明します。

**🌍 全マーカーを収める（Fit Bounds）**
右上の🌍ボタンを押すと、全マーカーが画面内に収まるよう、ズームと位置を自動調整します。ビューの外にマーカーがあっても、これで一気に全体を把握できます。

**縮尺を細かく選べる**
左上のズームボタン（＋/−）に加えて、左下のズームセレクターから16段階のプリセットを選べます。「Zoom 5（Country）」「Zoom 8（Wide Region）」「Zoom 13（City）」「Zoom 17（Street）」など目安のラベルつきで、一気に目的のスケールに移動できます。

**🔒 / 🔓 ズームロック**
地図を調べている最中に、うっかりスクロールでズームが変わってしまうことがあります。右上の🔒ボタンでズームを固定すると、パン（ドラッグ移動）はできてもズームは変わりません。固定すると、ズームボタン（＋/−）と左下のズームセレクターは非表示になります。

**マーカー詳細**
地図上のマーカーをクリックすると、右下に詳細パネルが開きます。ラベルの編集、メモの追記、削除ができます。ラベルは検索時の名前から好きな名前に変えられます。間違えてピンを立ててしまったときは、この画面から削除できます。

**📋 マーカー一覧**
右上の📋ボタンでマーカーの一覧を表示できます。あれこれ追加したあと、一覧から場所を確認して、不要なものはそのまま削除できます。一覧の場所名をクリックすると地図上のその場所に移動します。

**📤 📥 保存・読み込み**
調べた内容を残したいときは、右上の📤ボタンひとつでJSONにエクスポートできます。あとはリセットで全データを消して、次の調べ物をすぐに始められます。保存したデータは📥で読み込んで再開できます。

**🖨️ PDF書き出し**
右上の🖨️ボタンで、地図とマーカー一覧をPDFに書き出せます。旅行先が決まったとき、地図と場所名・メモをまとめて1枚に残せます。地図のみ、マーカー一覧のみを選ぶこともできます。

**🗑️ リセット**
右上の🗑️ボタンで全データを一括削除できます。前回の調べ物がそのまま残っていても、ボタン一つで消せるので、新しい調査をすぐに始めることができます。

## こんな場面で使っています

歴史書や本を読みながら、気になった地名を調べるのに使っています。中東の位置関係、ニュースに出てくる地名、最近では「ミラノ・コルティナ2026」オリンピックの競技会場の位置関係を調べてみました。

旅行プランを立てるときは、行きたい場所をどんどん検索してマーカーを追加していき、こまめに🌍ボタンで全体を収めて位置関係を把握する、という使い方を想定しています。エリアが掴めたらエクスポートして保存し、ズームインして細かい場所を確認する、という流れです。

アメリカ全州を俯瞰したいときはZoom 5（Country）が目安。検索した場所がどのあたりかわからないときは、Wide Region（広域）に切り替えると周辺の主要な地名が表示されてわかりやすいです。

検索は日本語でも動きますが、地名は英語の方が精度が高いです。「バグダッド」では見つからなくても「Baghdad」なら出てきた、ということがありました。たまに間違った場所にピンがついてしまっても、すぐに削除できます。

ライブデモでどんなものか試してみてください。気に入ったら、GitHubからクローンして自分用に作ってみてください。APIキーは不要ですが、Nominatimの利用規約でUser-Agentの設定が必要です。クローン後に `src/geocoding.ts` の `USER_AGENT` を自分のアプリ名とURLに変更してください。

```bash
git clone git@github.com:kkoisland/stable-context-map.git
cd stable-context-map
pnpm install
pnpm dev
```
