**stable-context-map** は、検索しても地図のビューポートが動かない地図アプリです。作ってから毎日のように使っています。今日もイラクや中東の地域の位置関係を調べるのに使いました。

通常の地図アプリでは、地名を検索するたびに地図が移動します。京都エリアを調べようと縮尺を調整しても、「清水寺」を検索すれば清水寺にズームインし、「金閣寺」を検索すれば金閣寺に移動します。旅行プランを立てるとき、地理の位置関係を調べるとき、こういったビューポートのリセットが繰り返されます。

- ライブデモ: https://www.kkoisland.com/stable-context-map/
- GitHub: https://github.com/kkoisland/stable-context-map

## できること

- 🔍 検索しても地図が動かない（検索結果はマーカーとして追加される）
- 📍 地図上のクリックでもマーカーを追加できる
- ✏️ マーカーにラベルとメモを編集できる
- 📋 マーカー一覧を右上に表示。選択・削除・地図上の場所へのジャンプができる
- 🌍 全マーカーが収まる範囲に自動調整（Fit Bounds）
- 🔒 ズームロックで誤操作を防止
- 📏 ズームセレクターで16段階のプリセットから選択
- 💾 localStorageに自動保存
- 📥📤 JSONでインポート／エクスポート
- 🖨️ PDFで書き出し

## アーキテクチャについて

このアプリはAPI Keyなしで動きます。OpenStreetMap / react-leaflet / Nominatim の構成については、別の記事で整理しています。

## 設計の核心：center を検索で更新しない

通常の地図アプリでは、検索後に `map.setView()` や `map.flyTo()` で地図を移動させます。stable-context-mapではそれをしません。検索結果はマーカーとして追加するだけで、地図の位置は変えません。

```ts
const handleSearch = async (query: string) => {
  const result = await searchLocation(query);
  const newMarker = createMarker(Number(result.lat), Number(result.lon), query);

  // 地図は動かさず、markerを追加するだけ
  setMarkers((prev) => [...prev, newMarker]);
};
```

`center` は React の state で持っていますが、検索のたびに更新しません。地図の視点はユーザーのものなので、検索という操作がそれを上書きすべきではない、という考えからです。

## 1件目だけ特別

ただし、最初のマーカー追加時だけは例外にしました。

```ts
if (markers.length === 0) {
  setCenter([newMarker.lat, newMarker.lng]);
}
setMarkers((prev) => [...prev, newMarker]);
```

最初の検索は「どのエリアを調べるか」というエリア設定として機能するからです。2件目以降は地図が動かないため、現在のビューの外にマーカーが追加されることもありますが、全マーカーが収まる範囲に自動調整するFit Bounds（🌍ボタン）で一度に表示できます。

## 特徴

**ズームロック**
地図を調べている最中に、うっかりスクロールでズームが変わってしまうことがあります。右上の🔒ボタンでズームを固定すると、パン（ドラッグ移動）はできてもズームは変わりません。縮尺を決めたら固定して、安心して地図を動かせます。

**全マーカーを収める（Fit Bounds）**
右上の🌍ボタンを押すと、全マーカーが画面内に収まるよう、ズームと位置を自動調整します。ビューの外にマーカーがあっても、これで一気に全体を把握できます。

**縮尺を細かく選べる**
左上のズームボタン（＋/−）に加えて、左下のズームセレクターから16段階のプリセットを選べます。「Country（国全体）」「City（市街）」「Street（通り）」など目安のラベルつきで、一気に目的のスケールに移動できます。ズームロック中は非表示になります。

**保存・読み込み**
右上の📤ボタンでJSON形式にエクスポート、📥ボタンで読み込みます。localStorageに自動保存されているので普段はそのままですが、別デバイスに引き継いだり、記録を残したいときに使えます。

**リセット**
右上の🗑️ボタンで全データを一括削除できます。前回の調べ物がそのまま残っていても、ボタン一つで消せるので、新しい調査をすぐに始めることができます。

## おわりに

「自分が使いたいものを作る」というのは、設計の判断軸がぶれないという点でよかったです。「検索でビューが動いてほしくない」という一点を守るために、center を検索で更新しない設計を選びました。そのシンプルな判断が、アプリ全体の一貫性につながったと思います。細かい挙動の不一致がまだ残っている部分もありますが、毎日使えるには十分な完成度になりました。

このアプリはNominatim（OpenStreetMapのジオコーディングAPI）を使っているため、大勢で同時に使うことには向いていません。興味があればGitHubからクローンして、ローカルで動かしてみてください。APIキーは不要ですが、Nominatimの利用規約でUser-Agentの設定が必要です。クローン後に `src/geocoding.ts` の `USER_AGENT` を自分のアプリ名とURLに変更してください。

```bash
git clone git@github.com:kkoisland/stable-context-map.git
cd stable-context-map
pnpm install
pnpm dev
```
