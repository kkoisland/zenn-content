2023年秋頃から、個人開発でAIを使い始めました。最初は「AIがあれば何でもできる」と思っていましたが、現実は違いました。試行錯誤を繰り返す中で気づいたのは、設計の重要性です。2025年秋頃から今のスタイルに落ち着いたので、本記事ではそのスタイルを紹介します。

## これまでの経緯

使い始めた当初、Shift-JISからUTF-8への大量変換をAIに頼もうとしました。しかし、方法を探してもらうこともできず、AIが変換することもできず、結局、過去のPerl経験を頼りに自分でコードを書きました。

それでも、迷ったときに相談できる存在ができたことは大きかったです。友達のように会話しながら開発することが多くなりました。

一方で、問題が解決しないまま同じ提案を繰り返される「AIループ」にハマることも多くありました。コードを直せば済む問題でも、別のライブラリをインストールするよう言われるなど、納得できないこともありました。

## 設計が大切

とにかく作り始めて、問題が出たらその都度AIに相談して直せば何とかなると思っていました。しかし、場当たり的に修正を重ねると、AIは過去の文脈と新しい方針を混ぜてしまい、保守不能なコードになってしまいます。

早く実装に入りたいと思っても、後で紹介するテンプレートに沿って、きちんとAIたちに相談して仕上げていくと、結果的にストレスなく時間も短縮できます。

また、実装を始めてちょっとした変更をするつもりでも、AIは大幅にコードを変更したり、冗長な書き方をすることがあるので、事前に変更の方針をしっかり確認することが大切です。

## 設計のフェーズごとにテストすることの重要性

QAをやっていた経験があり、テストの重要性は理解していたつもりでした。でも、開発を始めると「テストはいつでもできる」と、後回しにしがちでした。AIを使うようになって、フェーズごとにきちんとテストして設計の範囲内で直すことの重要性を実感しています。

AIは一般的な方法を提案してくれますが、やりたいことや、使い勝手は自分が一番よく知っています。テストをしながら、UX的にはどうなのか、なぜ一般的にはそのやり方がよくても、私が提案する方法のほうが理にかなっているか、など、徹底的に話しあって私なりの答えを出すようにしています。

個人開発でもブランチを切り、期待通りに実装できたときのみマージしています。やっている途中で冗長な点に気づいたときは、思い切ってそこまでの実装は捨て、前のフェーズに戻してやりなおす、ということもどんどんやっています。

## AI時代に何を学ぶか

自分でコードを書いていたときは、「まあここは私の限界だな」などと諦めていました。今は、適切に設計し、対話し、責任を持つことで、AIが強力な開発パートナーになっています。

ただ、「設計」や「テスト」は重要ですが、結局のところ、自分の能力を超えたものは作れないなあと気がつきました。AIは昨日言っていたことと、今日提案してきたことに一貫性がないこともあり、そこに気付けなければ、どんどん深みにハマってしまいます。

AIにコーディングを任せた分、技術や知識を学ぶ時間ができたと考え、特に自分の弱いところや新しいことは納得いくまで理解するようにしています。

ここからは、具体的なツールや方法を紹介します。

## 使っているAI

現在、2つのAIを併用しています。

### ChatGPT（Plus / GPT-5.2 $20）

**向いていること**
- 何でも気軽に相談できる
- レスポンスが早く、思考の壁打ちに向いている
- 技術以外の雑談・整理・文章化にも使いやすい

**苦手なこと**
- コーディングの際、リポジトリ全体構造を前提にした理解ができない
- 聞きたいことがあるとき、ファイルやコードスニペットを渡す必要がある
- 質問者の意図に寄せすぎて、設計の前提を深掘りしてくれないことがある

### Claude Code（Pro / Claude 3.5 $20）

**向いていること**
- リポジトリ全体を前提に理解してくれる
- 設定ファイルや構成を含めてまとめて作れる
- 技術的な是非をきちんと説明してくれる
- 必要ならNoと言ってくれる

**苦手なこと**
- トークン制限が厳しく、長く続けることはできない
- レスポンスが遅い

Claude Code一本化も検討しましたが、日常的な相談や思考整理にはレスポンスの早いChatGPTが欠かせず、この組み合わせに落ち着きました。

## 現在の開発フロー

現在は、以下のように役割を明確に分けて使っています。

| フェーズ | 内容 |
|---------|------|
| ① 調査 | ChatGPTに調べてもらい、議論し、最終判断は自分で行う |
| ② 設計 | Claudeに設計書案、ChatGPTと並行相談しながら固める |
| ③ タスク分解・実装案 | ②と同様に進め、実装の粒度に落とす |
| ④ 実装 | ファイル作成からすべてClaudeに任せる |
| ⑤ コード理解・学習・レビュー | 不明点はChatGPTで噛み砕いて理解する。納得できないときは、コードのコンテキストを知っているClaudeに確認する |
| ⑥ 変更・修正 | 使い勝手を私がチェックし、変更・修正はClaude |

コマンドライン操作や初期セットアップ、ライブラリの追加は自分でやっています。AIに任せると、意図しないバージョンが入ったり、余計な設定が入ることがあるためです。リファクタする場合は、E2Eテストを先に書くといいでしょう。

## テンプレート

AIとの開発で使っているテンプレートを紹介します。

Plan.md、TroubleShooting.mdなど複数ファイルに分けず、1ファイルにすべて入れています。1ファイルにまとめることで、ChatGPTに相談するときも渡しやすくなり、また、ファイルに残すことで、セッションが切れても続きから始められます。プロジェクトのリポジトリに入れるとGitHubでDiff確認時にノイズになるので、別の場所で保管しています。

:::details テンプレート構成（クリックで展開）
```
# ルール
（AIとの作業ルールを記載）

# 項目 ------------------
# 設計合意までのタスク
# 実装タスクリスト

# 設計書
## データ構造とデータフロー
## ワイヤーフレーム/画面構成
## プロジェクト構造
## 保存設計
## 状態管理
## モックデータ一覧
## ワークフロー
## アプリ設定 / 将来拡張ポイント

# 技術スタック
# 開発ルール
# レポ＆URL基本情報

# 人間エリア
# 人間ログ ------------------
# ログ ------------------
```
:::

技術スタックや開発ルールなど、プロジェクトをまたがって使うものは最初から指定しています。「人間エリア」と「ログ」には、私の必要なメモを入れています。
