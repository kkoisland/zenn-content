「Amazon Web Services」について。

## AWS コンテナサービス
`Amazon Elastic Compute Cloud(EC2)` -  仮想マシン（インスタンス）を提供する。ユーザーは、オペレーティングシステムの選択、ネットワーキング、セキュリティ設定、ストレージ管理など、インスタンスの全ての側面を管理する。最も柔軟性があり、任意のアプリケーションをホストできるが、管理が複雑。
`Amazon Elastic Container Service(ECS) `- コンテナオーケストレーション。コンテナベースのアプリケーションを管理する。コンテナをEC2インスタンス上で直接実行するか、またはFargateを使用してサーバレス環境で実行するかを選択できる。
## サーバーレスコンピューティングオプション
`AWS Lambda` - イベント駆動の短期実行タスクに適する
`Amazon Fargate` - コンテナベースの長期実行アプリケーションに適する
## AWS でのネットワーク
`Amazon Virtual Private Cloud` ユーザーがAWSクラウド内に仮想ネットワークを構築できる環境を提供する
## ストレージ
`Amazon S3`スケーラブルなオブジェクトストレージサービスで、任意の量のデータを保存し、世界中のどこからでもアクセスできる。データはオブジェクトとして保存されるため、オブジェクトストレージと呼ぶ。保存されるファイルは単にデータとしてではなく、メタデータと識別子を含む包括的なオブジェクトとして保存される。
`Amazon Elastic Block Store(EBS)`EC2インスタンス用の高性能なブロックレベルストレージに特化している。
## データベース
`Amazon DynamoDB`ウェブサイト、モバイルアプリケーション、ゲーム、IoTデバイスなど、様々なアプリケーションにおけるデータストレージと管理のニーズに対応するための効率的なソリューションを提供する。NoSQLデータベースサービス。
## 監視と管理
`Amazon CloudWatch` AWS環境の健全性とパフォーマンスを継続的に監視し、最適化するための重要なツール。運用と管理に焦点を当てる。
## ネットワーキングとコンテンツ配信
`Elastic Load Balancing（ELB）`AWSの負荷分散サービス、入ってくるアプリケーショントラフィックを複数のターゲット（例えば、Amazon EC2インスタンス、コンテナ、IPアドレス）に自動的に分散する機能を提供する。
